# Security Policy

| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please follow these steps to report it responsibly:

1. **Confidential Reporting**: Send an email to [szymon.goral@icloud.com] with a detailed description of the issue, including steps to reproduce, potential impact, and any relevant evidence (logs, screenshots, etc.).
2. **Acknowledgement**: We will acknowledge receipt of your report within 48 hours.
3. **Investigation and Resolution**: Our team will promptly investigate the issue. You can expect regular updates (at least every 7 days) until the issue is resolved.
4. **Coordinated Disclosure**: Please refrain from publicly disclosing the vulnerability until a patch is available or you have received explicit permission from the maintainers.

Thank you for your contribution to the security of our project.

## Security Updates

Verified vulnerabilities will be addressed in the next available release. We strongly recommend that you regularly update to the latest version to benefit from critical security patches.

## Responsible Disclosure

We adhere to a responsible disclosure process and value the efforts of security researchers. Your cooperation in reporting vulnerabilities privately helps us ensure a secure environment for all users.
